    About

    Looking for a simple authentication solution for your Bukkit enabled Minecraft server? Don't want to deal with lots of configuration or daunting commands? Then look no further. This is exactly what BitAuth is all about. It is designed to be run on servers that operate with the online-mode flag set to false.

    For a list of permission nodes and commands see the included plugin.yml file.

    A word on passwords

    At no point does this plugin ever store passwords. This uses a one-way digest algorithm with a variable, private salt. It creates a random salt during registration via Java's SecureRandom. This salt is used to create the password hash and is stored along with the resultant hash. This salt is used to regenerate the same hash when a player is logging in after registration. If the resultant hash matches the stored hash, the password is correct.

    Features

    • Player registration and unregistration
    • Logging in and out (can be forced by admin)
    • Pre-login interaction, chat and command prevention
    • 30 minute extended login period (only applies to players logging in within this period from the same IP address)
    • Login time limit, default is 2 minutes - kicks after this period
    • Toggled per-player IP address checking (if enabled, prevents users from logging in from new IP addresses)
    • Fully featured whitelist system
    • Permission support (currently limited to PEX)
    • Automatic database table generation (must be properly connected)
    • Password changing by player, password resets by administration
    • Players not marked as logged in will not take damage
    • Disable login for specific players by jumbling their passwords
    • System to prevent name-spoofing for users on their 30 minute extended login period
    • Password hash salting and iterations
